Práticas de escrita front-end

Padrões e boas práticas para escrever códigos HTML, CSS e Javascript

Quando definimos um padrão de escrita de código, queremos que o código seja uniforme. Isso quer dizer que o código precisa parecer que foi escrito por uma única pessoa. Assim, conseguimos manter a manutenção mais simples, garantindo que o projeto seja flexível e escalável de forma consistente.

Estes padrões são usados por todos os times de desenvolvimento de produtos da Locaweb. Você pode também colaborar com essas boas práticas discutindo e melhorando nosso projeto no Github.

Pode ser que você não siga algumas dessas práticas ou utiliza outras diferentes. Infelizmente não há como seguirmos todos os gostos pessoais de cada um por isso precisamos escolher apenas um.
